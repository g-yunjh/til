### 1. 물리 서버

#### ▸ 서버 외관과 설치 장소

- **서버의 외형**은 크게 세 가지 종류로 나뉘어:
    
    - **랙 서버 (Rack Server)**: 표준화된 규격(1U, 2U 등)의 박스 형태. 수십~수백 대를 수직으로 쌓을 수 있어 데이터센터에서 주로 사용됨.
        
    - **타워 서버 (Tower Server)**: 일반 데스크탑처럼 생긴 형태. 작은 사무실 환경에 적합.
        
    - **블레이드 서버 (Blade Server)**: 초소형 모듈형 서버. 블레이드 섀시 안에 여러 대의 서버가 꽂혀 있는 구조. 고밀도 서버 환경(한정된 공간에 더 많은 서버)에 적합.
        
- **설치 장소**는 일반적으로 IDC(Internet Data Center) 또는 자체 서버룸.
    
    - 전력공급, 냉방, 방진(먼지 방지), 보안, 네트워크 연결 등 인프라 설비가 필요함.
        
    - 전원이 이중화되어 있고, UPS(무정전 전원 장치)나 발전기가 있어야 함.
        

---

#### ▸ 서버 내부 구성

- 내부는 여러 핵심 부품으로 구성:
    
    - **CPU**: 중앙처리장치
        
    - **메모리 (RAM)**: 데이터 임시 저장공간
        
    - **스토리지 (HDD/SSD)**: 영구 저장공간
        
    - **네트워크 카드 (NIC)**: 외부와의 통신 담당
        
    - **마더보드**: 모든 부품이 연결되는 중심 회로
        
    - **전원공급장치 (PSU)**: 부품에 안정적으로 전원 공급
        
    - **쿨링 시스템**: 열 방출을 위한 팬 또는 수냉식 냉각장치
        

---

### 2. CPU

- **서버의 핵심 연산 장치**, 사람으로 치면 두뇌 역할.
    
- 주요 성능 지표:
    
    - **클럭 속도 (GHz)**: 1초에 처리할 수 있는 연산 횟수
        
    - **코어 수**: 동시에 처리할 수 있는 작업 수 (멀티코어는 병렬 처리에 유리)
        
    - **쓰레드 수**: 코어 내 논리적 처리 단위
        
    - **캐시 메모리 (L1, L2, L3)**: CPU가 자주 쓰는 데이터를 임시로 저장해 속도를 높이는 장치
        
- 서버용 CPU는 일반 데스크탑용보다 더 많은 코어, 더 높은 신뢰성, 에러 검출 기능(ECC-오류 정정 코드 지원 등)을 갖춤.
    
    - 예: **Intel Xeon**, **AMD EPYC**
        

---

### 3. 메모리

- **작업 중인 데이터를 일시적으로 저장하는 고속 기억장치**.
    
- 서버에서는 **대용량 메모리**가 필수:
    
    - 다수의 사용자 요청 처리
        
    - 대규모 데이터 처리 (예: DB, 웹 트래픽 등)
        
- **ECC 메모리 (Error-Correcting Code Memory)** 사용:
    
    - 데이터 오류를 자동으로 감지 및 수정
        
    - 서버의 안정성과 데이터 무결성을 높여줌
        

---

### 4. I/O 장치

#### ▸ 하드 디스크 드라이브 (HDD)

- **서버의 데이터 저장소**, 데이터를 영구적으로 저장.
    
- 전통적인 HDD는 회전하는 디스크를 사용해 데이터를 읽고 씀 → 비교적 느림.
    
- 서버에서는 다음과 같은 방식으로 보완:
    
    - **SSD(반도체 메모리 칩)** 병행 사용: 빠른 응답 속도 (DB 캐시, OS 저장용)
        
    - **RAID 구성**: 여러 개의 디스크를 묶어 성능 및 신뢰성 향상
        
        - RAID 0: 속도 ↑ -> **속도 중심**
            
        - RAID 1: 복제(미러링)로 안정성 ↑ -> **복제 중심**
            
        - RAID 5/6/10: 속도와 안정성의 균형 -> **균형형**
            

---

#### ▸ 네트워크 인터페이스

- **서버와 외부(인터넷/내부망)를 연결하는 장치**.
    
- 대부분의 서버는 이더넷 기반 NIC(Network Interface Card)를 사용.
    
    - 1Gbps, 10Gbps, 25Gbps 등 속도 다양
        
    - 서버마다 2개 이상 장착해 **이중화** 및 **로드 밸런싱** 구성 가능
        
- 고속 처리 요구 시, **광케이블 기반 네트워크 카드** 사용 (예: SFP+)
    

---

#### ▸ I/O 제어

- CPU와 I/O 장치 간 데이터 전송을 효율적으로 제어.
    
- 보통은 **DMA(Direct Memory Access)** 기능 사용:
    
    - CPU가 개입하지 않고, 메모리와 장치 간 직접 데이터 송수신 → CPU 부하 감소
        
- 이를 통해 **입출력 병목** 최소화, 시스템 전체 성능 향상
    

---

### 5. 버스

#### ▸ 대역 (Bandwidth)

- **버스 대역폭**은 데이터가 한 번에 오갈 수 있는 양을 의미.
    
    - 마치 데이터가 지나가는 고속도로의 차선 수 같은 느낌.
        
    - 예: 32bit 버스보다 64bit 버스가 더 많은 데이터를 동시에 처리 가능
        

#### ▸ 버스 대역

- CPU, 메모리, I/O 장치 간 데이터 통신을 연결하는 **경로(버스)**의 총 처리량.
    
- 종류:
    
    - **프론트 사이드 버스 (FSB)**: CPU와 메모리 연결
        
    - **백플레인 버스**: 장치 간 연결 (스토리지, 네트워크 카드 등)
        
- 최신 서버는 **시리얼 버스 구조 (데이터를 한 줄로 차례차례 빠르게 보내는 통신 방식) (예: PCIe)** 사용 → 더 빠르고 안정적
    
- 병목 현상 방지를 위해 **멀티 채널**, **고속 버스 아키텍처** 적용
